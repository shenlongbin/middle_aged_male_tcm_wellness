# Build Plan

## Current Build Phase

Pre-implementation to MVP front-end.

## Recommended Stack

- Static HTML/CSS/JavaScript for the first version
- Optional lightweight form backend later
- Basic analytics such as Plausible, Umami, or GA4

## Build Sequence

1. Finalize homepage copy and information architecture
2. Create a static landing page MVP
3. Add topic sections and conversion form
4. Add analytics and CTA event tracking
5. Deploy to a simple hosting platform
6. Distribute through one or two acquisition channels

## File Plan

- `index.html`: homepage and MVP landing structure
- `styles.css`: layout and visual system
- `script.js`: interactions and simple tracking hooks
- `docs/`: strategy and validation documents

## Engineering Notes

- Favor readability and speed over abstraction
- Keep asset footprint small
- Make the first version mobile-friendly
- Avoid unnecessary framework complexity before validation

## Definition of MVP Build Done

- Homepage communicates the value proposition clearly
- User can understand the offer within seconds
- At least one conversion action is visible and working
- Site is deployable as a static project
